# 게임 업데이트 성과 보고서 생성 스킬 v2

## 개요
어떤 게임이든 업데이트 현황 엑셀 파일을 첨부하면 HTML 형식의 성과 분석 보고서를 생성함.
게임명·업데이트명·컨텐츠 용어는 엑셀 원본 표기를 그대로 따르며 임의 추측하지 않음.

## 실행 절차

### 0단계 — 사전 확인 (항상 실행)
파일을 읽기 전, 반드시 ask_user_input_v0 도구를 사용해 아래 두 질문을 순서대로 띄운다.

**질문 1 — 작성자 이름**
- 질문: "작성자 이름을 입력해 주세요."
- 타입: 단답형 자유 입력

**질문 2 — 서식 사용 여부**
- 질문: "엑셀 파일 안에 표나 차트의 서식이 지정되어 있다면 해당 서식으로 작성하겠습니다. 서식이 지정되어 있나요?"
- 선택지:
  - A1. 예 → 파일 내 서식을 참고하여 작성
  - A2. 아니요 → 알아서 작성
  - A3. 일부만 있을 경우 직접 입력 → 자유 입력

| 답변 | 동작 |
|------|------|
| A1 (예) | 1단계에서 openpyxl로 전체 시트의 표·차트 서식을 읽어 HTML에 반영 |
| A2 (아니요) | 서식 읽기 생략. 아래 "차트 타입 결정 원칙 — 기본값" 적용 |
| A3 (직접 입력) | 입력된 내용에 해당하는 시트·항목의 서식만 읽어 반영. 나머지는 기본값 적용 |

> ⚠️ 이 단계는 엑셀 데이터와 무관하게 항상 실행한다. 자동 판단 가능 여부와 관계없이 생략하지 않는다.

### 1단계 — 파일 읽기 및 구조 파악

| 확인 항목 | 방법 |
|-----------|------|
| 게임명 / 업데이트명 | 파일명 또는 시트 내 텍스트 참조 |
| 분석 대상 날짜 / 차수 | 데이터의 날짜 범위 확인 |
| 분석 유형 | 아래 "분석 유형 판별" 참조 |
| 시트 목록 | 접속 / 매출 / 게임 컨텐츠(0~N개) |
| 비교 대상 업데이트 목록 | 데이터에 포함된 이전 업데이트명 확인 |

**분석 유형 판별**

| 유형 | 판별 기준 | 비교 방식 |
|------|-----------|-----------|
| 업데이트 단위 분석 | 복수 업데이트의 동일 차수 데이터가 나란히 존재 | 업데이트 간 횡단 비교 |
| 업데이트 전후 분석 | 하나의 업데이트 전후 일별 추이 데이터가 존재 | 1일차: 이전 7일 일별 / 1주차: 이후 7일 누적 비교 |
| 4주 시계열 분석 | 4주치 주간 데이터가 있을 경우 | 시계열 라인/막대 차트 |

**서식 읽기 (0단계 답변이 A1 또는 A3일 때만 실행)**

```python
import openpyxl

wb = openpyxl.load_workbook('파일경로.xlsx')  # read_only=False 필수

for ws in wb.worksheets:
    for chart in ws._charts:
        print(type(chart).__name__)
        print(chart.title)
        for i, s in enumerate(chart.series):
            print(f'  시리즈{i}:', s.title)
            print(f'    val 참조범위:', s.val)
            print(f'    cat 참조범위:', s.cat)
```

| 엑셀 차트 타입 | HTML 구현 방식 |
|----------------|----------------|
| BarChart | SVG 세로/가로 막대 차트 |
| LineChart | SVG 폴리라인 차트 |
| DoughnutChart | SVG 도넛 차트 (stroke-dasharray 기법) |
| PieChart | SVG 파이 차트 |

### 2단계 — 핵심 수치 계산

엑셀에 존재하는 항목만 계산. 없는 데이터는 절대 추측하지 않음.

**[접속]**
- 계정수(DAU) / NPU 현재 vs 직전 업데이트 증감
- 유입 구성비: 신규 / 복귀 / 기존 비율
- 국가별 접속 비중 (데이터가 있을 때만)

**[매출]**
- 매출액(VAT 제외) / PU / ARPPU 현재 vs 직전 업데이트 증감
- 국가별 매출 비중 (데이터가 있을 때만)
- 상품별 구매 현황 (데이터가 있을 때만)

**[게임 컨텐츠]**
- 게임 컨텐츠 시트가 1개 이상 있을 경우, 해당 데이터만 기반으로 분석
- 시트가 없으면 3번 섹션 전체 생략

### 3단계 — HTML 보고서 생성
아래 보고서 구조 및 스타일 규칙을 따라 HTML 아티팩트로 생성.

### 4단계 — 검토 요청
보고서 생성 후: "추가로 분석이 필요한 항목이 있으시면 말씀해주세요." 한 줄만 출력.

## 보고서 구조

**제목**
- `{게임명}_{업데이트명} {N}일차 현황`
- `{게임명}_{업데이트명} {N}주차 현황`

**상단 고정 정보**
- 작성일 (보고서 생성 날짜) / 작성자 (0단계에서 입력받은 이름)

**Summary 섹션 (최상단, 본문 이전)**
- 핵심 내용을 불릿(•) 3~5줄 요약
- 수치 필수 포함, 직전 업데이트 대비 증감 명시
- 말투: 명사형 종결 (~임, ~함, ~됨)

**본문 섹션**

| 번호 | 섹션명 | 포함 내용 |
|------|--------|-----------|
| 1 | 접속 현황 | DAU/NPU 전체 현황, 유입 유형(신규·복귀·기존) 비율, 국가별(데이터 有 시) |
| 2 | 매출 현황 | 매출액(VAT제외)·PU·ARPPU, 국가별·상품별(데이터 有 시) |
| 3 | 컨텐츠 현황 | 게임 고유 컨텐츠 (신서버·던전·이벤트 등, 시트 有 시만) |

## 차트·표 규칙

**차트 타입 결정 원칙 — 기본값 (A2 또는 서식 없는 항목)**

| 데이터 성격 | 기본 차트 타입 |
|-------------|----------------|
| 업데이트 간 수치 비교 | BarChart (묶음 막대) |
| 시계열 추이 | LineChart |
| 비율/구성비 | DoughnutChart 또는 수평 누적 막대 |
| 유입 유형 비율 | 수평 누적 막대 (신규/복귀/기존) |

**수치 표기 규칙**

| 항목 | 규칙 |
|------|------|
| 비율 | 엑셀 원본 소수점 (0.18xx) → ×100 → % 표기 |
| 매출액 | ₩, 억 기준 소수점 둘째 자리까지 (xx.yy억) |
| 증감 | ▲ or ▼ + 비율% + (수치) — 예: ▼21.2%(24,970) |
| 강조 수치 | `<strong><u>굵게 밑줄</u></strong>` |

## 용어 정의

| 약어 | 정의 |
|------|------|
| DAU | Daily Active User — 일별 활성 유저 (접속 계정수) |
| NPU | NC Player Unit — 본인 인증 기준 중복 제거 유저 단위 |
| PU | Paying User — 결제 계정수 |
| ARPPU | Average Revenue Per Paying User — 결제 계정당 평균 매출 |
| 신규 유입 | 업데이트 이후 처음 유입된 계정/NPU |
| 복귀 유입 | 업데이트 이전 2주간 미접속했던 계정/NPU |
| 기존 유입 | 신규·복귀를 제외한 나머지 계정/NPU |

## 주의사항
- 데이터에 없는 내용은 절대 추측하여 작성하지 않는다
- 게임명·업데이트명은 파일/시트에서 확인한 값을 그대로 사용한다
- 국가별 분석은 엑셀에 해당 시트/컬럼이 있을 때만 포함한다
- 게임 컨텐츠 시트가 없으면 섹션 3 전체를 생략한다
- 서식 읽기는 0단계 답변이 A1 또는 A3일 때만 실행한다
- 차트 서식 읽기 시 values_only=True로 로드하면 차트가 감지되지 않음 — 반드시 일반 모드(read_only=False)로 로드
- 전 업데이트 대비 수치가 크게 변화한 경우 인사이트 박스에 반드시 언급한다
- 인사이트는 수치 나열이 아닌 "왜 이런 결과가 나왔는가" 관점으로 작성한다
